# 实习能量站 · 方案说明

> 全文约 950 字。Demo 在线访问：`https://<your-name>.github.io/intern-energy-station/`

## 一、问题诊断

新一批 20 名实习生入职后，我们观察到三类典型痛点：

1. **导师侧**：业务部门导师凭经验带教，节奏因人而异；周反馈口头化、留痕少；新人多人时易顾此失彼。
2. **实习生侧**：4 周培养路径不透明，新人「不知道下周该干嘛」，频繁私聊 HR；输出动作（周报/复盘）无结构、写起来痛苦。
3. **协同侧**：HR 同时承担招聘协同职责，却缺一份跨阶段的「批次全景 + 适岗评估」数据，难以回答业务老板「这批人怎么样」，也难凭印象判断「谁可以留用」。

根因可以归纳为 **三缺**：缺标准化路径、缺数据化进度、缺 AI 化辅助。

## 二、方案设计

围绕 **「一条路径 × 三个角色 × 五个 AI 节点」** 设计「实习能量站」智能看板：

- **一条路径**：把零散培养动作沉淀为 4 阶段 SOP：入职准备 → 业务熟悉 → 独立产出 → 答辩转正，每阶段配标准任务清单与产出。
- **三个角色**：实习生 / 导师 / HR（含招聘协同）各自独立工作台，但共享同一份成长数据。顶部「演示切换」让评审一键体验 3 个视角。
- **五个 AI 节点**：
  1. *AI 导师答疑*（实习生）：注入岗位/团队/导师上下文，随时问；
  2. *AI 周报助手*（实习生）：输入几条要点 → 一键生成「成果/问题/下周计划」三段式 → 可继续编辑；
  3. *AI 反馈助手*（导师）：输入观察事实 → 自动按 **STAR** 起草反馈，存档可追溯；
  4. *AI 批次报告*（HR）：以全员数据快照为输入，一键生成「概况 / 对比 / 风险 / 亮点 / 建议」五段式总结，支持复制 / 下载 .md；
  5. *AI 转正建议*（HR / 招聘协同）：综合「技能 60% + 进度 30% + 反馈 10%」打 ABC 分级，AI 输出留用建议与试用岗位推荐。

辅以「**风险预警引擎**」：自动识别任务逾期、周报缺交、进度低于 30%、技能均分低于 60、零反馈五类信号，HR 视图按高/中/低三档分级展示，从「被动找问题」变「问题主动找上来」。

## 三、AI 工具选型理由 & 关键配置

- **模型选型**：选用 **DeepSeek Chat (`deepseek-chat`)**——中文意图理解准确、价格友好、支持标准 OpenAI 协议，便于前端直连。
- **接入方式**：浏览器直连 `https://api.deepseek.com/v1/chat/completions`，Key 仅保存在用户本机 `localStorage`，零后端、零运维。
- **降级保护**：未配置 Key 或调用异常时自动回落到 **本地 mock 模板**（5 类场景化回答），保证演示与离线评审依然完整可用。
- **Prompt 工程**：每个 AI 节点都注入**结构化角色与数据上下文**（岗位、导师、阶段、技能向量），并明确产出模板（STAR / 三段式 / 五段式），把不确定的对话变成可复用的结构化产出。

## 四、关键技术配置

- **栈**：React 18 + TS + Vite + Tailwind + Zustand(persist) + Recharts。
- **路由**：`HashRouter` 规避 GitHub Pages 子路径刷新 404；`404.html` 双保险。
- **数据**：20 名实习生 + 5 名导师 + 3 周历史周报 + 12 条导师反馈，全部 mock 即用，**所有写操作通过 zustand 持久化到 localStorage**，刷新不丢失，便于演示。
- **部署**：GitHub Actions 工作流（[`.github/workflows/deploy.yml`](../.github/workflows/deploy.yml)）push main 即自动构建并发布到 GitHub Pages。

## 五、迭代记录与效果评估

- v0.1 → v0.5：搭建路径模型与数据；
- v0.6：补齐 4 角色页面与 AI 五节点；
- v0.7：新增风险预警引擎与批次趋势；
- v1.0：完善 mock 数据、降级保护与文档。

**预期效果**：
- 导师写周反馈耗时由 ~20 分钟降至 ~5 分钟；
- 实习生写周报耗时由 ~40 分钟降至 ~10 分钟；
- HR 回答业务老板「批次情况」由 30 分钟手动汇总降至 30 秒一键生成；
- 招聘协同判断「谁可留用」由凭印象升级为有数据支撑的 ABC 分级。
